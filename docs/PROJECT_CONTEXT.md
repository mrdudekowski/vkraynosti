Каноническая «карта местности» для разработчиков и аналитики: продукт, стек, зафиксированные решения и **инженерный backlog по проблемным местам в коде**. Детальные чек-листы контента и продакшена — в [PRODUCTION_READINESS.md](./PRODUCTION_READINESS.md); структура репозитория — в [ARCHITECTURE.md](./ARCHITECTURE.md). Правила кодирования — в `.cursor/rules/vkraynosti.mdc`.

---

## Продукт

SPA туристического агентства **Вкрайности**: главная с сезонным UX, каталог туров по сезонам, страница тура, безопасность, политика конфиденциальности, заявка на тур (клиентская валидация). Деплой ориентирован на **GitHub Pages** с базовым путём `/vkraynosti/` (`vite.config.ts`, `basename` роутера, `playwright.config.ts`).

## Стек (факт из репозитория)

| Слой | Технологии |
|------|----------------|
| UI | React 19, TypeScript, Vite 7, Tailwind CSS 3 |
| Роутинг | React Router 7 (`createBrowserRouter`, lazy-страницы) |
| Состояние | React Context (`Modal`, `Season`, навбар/меню сезонов) — **без Redux** |

| Мета | `react-helmet-async` (`PageMeta`) |
| Валидация | Zod (`src/validation/tourRequestSchema.ts`) |
| Иконки | FontAwesome React (+ осознанные исключения, см. PRODUCTION_READINESS) |
| Тесты | Vitest (юнит/константы/модалка), Playwright (`e2e/`) |
| CI | `.github/workflows/deploy.yml` — audit, lint, unit tests, build, Pages (**e2e в CI не запускаются**) |

## Зафиксированные продуктовые решения (ранее «открытые вопросы»)

| Тема | Решение |
|------|---------|
| Плейсхолдеры в контактах и смежном контенте | **Намеренно** до финальной вычитки; не считать багом текущего этапа. |
| Канонические URL и `SITE_URL` | Приводятся к целевому виду **непосредственно перед продакшеном** (единая правка констант/конфига и проверка OG/ссылок). |
| Аналитика и cookies | **Нужны к продакшену**: учёт посещаемости и **согласие на cookies** (юридически корректный баннер/настройки). Базовый вариант аналитики — **Яндекс.Метрика**; к альтернативам или гибриду открыты. |
| Отправка и хранение заявок на тур | **Проектируется и реализуется перед деплоем** (сейчас в UI нет реального endpoint; см. модалку заявки). |

## Архитектура в одном абзаце

`main.tsx` оборачивает приложение в Lenis → ошибки → Helmet → сезон → модалки → `RouterProvider`. Корневой маршрут ренерит `Layout` с `Outlet`; страницы сезонов, тура, safety/privacy и 404 подгружаются **lazy**. Данные туров/команды/безопасности — типизированные модули в `src/data/`, строки UI и пути — `src/constants/`, визуальные токены — `tailwind.config.ts` и `index.css`.

## Инженерный backlog: проблемные места в коде

Приоритеты: **P0** — сделать до любого публичного продакшена; **P1** — качество и SSoT; **P2** — процесс и документация.

### P0 — безопасность и шум в проде

- [ ] **`src/components/shared/PageMeta.tsx`** — удалить отладочные вызовы `fetch('http://127.0.0.1:7539/ingest/...')` в трёх `useEffect` (регионы `#region agent log`). В продакшене это лишние запросы, шум в консоли и непрозрачная утечка метаданных страницы в локальный сервис.

### P1 — соответствие правилам проекта и предпродакшен

- [ ] **`src/pages/Home.tsx`** — вынести локальные числовые константы анимации сетки туров (`GRID_*_MS` и т.п.) в `src/constants/*`, чтобы тайминги оставались в одном стиле с остальным home/баннером.
- [ ] **`src/constants/routes.ts`** — устранить риск расхождения: `buildTourDetailPath` вручную собирает `/tours/...`, а шаблон маршрута живёт в `ROUTES.TOUR_DETAIL`; иметь один источник сегментов или генератор из шаблона.
- [ ] **Яндекс.Метрика + cookie consent** — внедрить загрузку счётчика после согласия (или режим, соответствующий выбранной юрисдикции), хранение выбора пользователя, связь с текстом политики в `UI`/странице privacy.
- [ ] **Заявка на тур** — заменить заглушку отправки реальным каналом (API, serverless, почтовый сервис) и обработку ошибок; секреты только через env в CI/хостинге.

### P2 — качество поставки

- [ ] **Playwright в CI** — отдельный workflow или job с кешем браузеров; либо явно задокументировать в [README.md](../README.md), что e2e только локально.
- [ ] **README** — кратко: `base`/`basename`, генерация медиа (`npm run generate:*`), правило новых ассетов через `src/constants/images.ts`.

### Уже задокументировано в PRODUCTION_READINESS (не дублировать задачами здесь)

Контент (безопасность JSON, placeholder-медиа, favicon, robots/sitemap), `docx` в devDependencies, прочие контентные чекбоксы — вести в [PRODUCTION_READINESS.md](./PRODUCTION_READINESS.md).
