## Профилирование в Chrome (главная и страница тура)

Инструменты: **React DevTools → Profiler** и вкладка **Performance**. Сборка: `npm run dev`, открыть `http://127.0.0.1:5173/vkraynosti/` (или ваш dev URL с `basename`).

### React Profiler

1. Установить [React Developer Tools](https://react.dev/learn/react-developer-tools).
2. Вкладка **Profiler** → включить запись (**Record**) → выполнить сценарий (скролл главной, смена фокуса ворот, скролл страницы тура с программой).
3. Остановить запись. Смотреть **рендеры** и **время коммита**: что перерисовывается слишком часто без смены данных.
4. Для сравнения «до/после» оптимизаций сохранить скрин или заметить **количество коммитов** на одно действие.

**Ожидаемое по коду:** скролл не должен вызывать `setState` на каждый кадр в `useScrollScrubFade` / `useHomeSkyParallax` (там правки DOM через `style`). Контекст навбара главной (`HomeNavbarChromeContext`) обновляется при изменении **квантованного** состояния хрома, не на каждый пиксель скролла.

### Performance (Main thread)

1. Вкладка **Performance** → **Record** → тот же сценарий (скролл 5–10 с, открытие модалки при необходимости).
2. Смотреть **Main** — длинные задачи (Long tasks **> 50 ms**), частоту **Layout / Paint**.
3. При подозрении на сеть — колонку **Network** в том же сеансе или отдельно **Network** с throttling (Slow 3G).

### Связь с кодом (что уже учтено в проекте)

- **`React.memo`** на тяжёлых «листьях» с стабильными props: `TourCard`, hero карусели, баннер сезона на воротах, `TourDetailHero`, `TourDetailGallery`, фон сезона.
- **`useMemo` / `useCallback`** на странице тура для галерейных URL, колбэка открытия фото и постеров видео — чтобы не инвалидировать `memo` дочерних компонентов при каждом ререндере (в т.ч. при обновлении счётчика раскрытия программы).
- **`useTourProgramScrollReveal`**: обновление состояния только при **смене** раскрытого шага (`setScrollRevealedCount` с пропуском равного значения).

## 2. Сеть на странице тура (видео и медиа)

Сценарий **`e2e/tour-detail-performance.spec.ts`** проверяет ожидаемое поведение (например, число video-ресурсов и `preload` после скролла). Это **регрессионный эталон**, а не абсолютный бюджет байтов.

- При изменении галереи или видео — прогон `npm run test:e2e:perf` и при необходимости обновление порогов в спецификации **осознанно**, с комментарием в PR.

## 3. Main thread / CPU (интерактив)

| Что измерять | Инструмент | Заметки |
|--------------|------------|---------|
| Долгие задачи (>50 ms) | Performance → Main thread в Chrome | главная, страница тура, открытие модалки |

## 4. Покрытие тестами критичных путей

- Unit: `npm test`, при необходимости `npm run test:coverage` — смотреть не только процент, но и **критические модули** (маршруты, валидация форм, разметка галереи).
- E2E: при необходимости локально (`npm run test:e2e`, `npm run test:e2e:perf`). В CI не запускаются — только ручной или отдельный прогон у разработчика.

## 5. Как зафиксировать целевые метрики (процесс)

1. Снять baseline на текущем `main` (сеть + один профиль CPU для главной и тура).
2. Заполнить таблицы в этом файле или в wiki/issue tracker.
3. Установить **верхние границы** там, где бизнес готов их принять (например, LCP для главной, суммарный вес initial JS).
4. В CI на push в `main`: **audit** (уровень high+), **lint**, **unit**, **build**, деплой. Lighthouse и e2e — по необходимости локально.

## Связь с кодом

- Разбиение бандла: `vite.config.ts` → `build.rollupOptions.output.manualChunks` (в т.ч. отдельные чанки `@fortawesome/free-solid-svg-icons`, `free-brands-svg-icons`, остальной `@fortawesome`).
- Импорты иконок — по файлам вида `@fortawesome/free-solid-svg-icons/faXxx` (не из общего `index`), чтобы бандлер не тянул лишнее при изменениях цепочки импортов.
- Модалки **«заявка на тур»** и **карточка команды** подключаются лениво из `Layout` (`React.lazy`), чтобы не раздувать начальный JS до первого открытия.
- Плавный скролл: **Lenis** + нативный `requestAnimationFrame` (пакет `motion` не используется — меньше JS на клиенте).
- Главная: под сеткой туров три секции грузятся **лениво** (`src/pages/Home.tsx`); плейсхолдер — `HomeBelowFoldSuspenseFallback`, высота `min-h-home-below-fold-suspense` в теме Tailwind.
- Модалки заявки и участника команды: пока грузится чанк — `ModalLazyChunkFallback` в `Layout.tsx` (`aria-label` из `UI.modal.lazyChunkLoadingLabel`).
- Базовый URL и кэширование: `base` в Vite, заголовки на стороне GitHub Pages (по умолчанию).
