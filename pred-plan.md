# Pred-plan: баннер «В другой сезон» (градиент, мягкие переходы, Satyr SP)

Документ фиксирует согласованное направление до внедрения в код. Реализация — отдельными коммитами по фазам.

## 1. Цель

Заменить статичное фоновое изображение в секции переключения сезона на главной (`Home.tsx`, блок с `SeasonSwitcher` и подписью «В другой сезон») на **стильный градиентный баннер** с:

- **десятью вертикальными колонками** (10 букв слова «Вкрайности»);
- в каждой колонке — **короткий сезонный видеофрагмент** (зима только из зимних туров и т.д.);
- **мягкими переходами** между фазой «видео» и фазой «буква»: fade in / fade out **без резкого мигания**, с **виньетированием / затенением по краям колонки** и при необходимости лёгким общим градиентным оверлеем на секцию;
- типографикой букв — **Satyr SP Basic** ([`fonts/SatyrSP Basic.otf`](fonts/SatyrSP%20Basic.otf)), визуально отдельно от wordmark в навбаре (Dela Gothic One), но в палитре и настроении сайта.

## 2. Охват по viewport

| Условие | Поведение |
|--------|-----------|
| **Mobile** (ниже `md`, 768px) | Анимация колонок и видео **не нужна**. Статичный fallback: градиент сезона + текст/переключатель без сетки из 10 клипов (детали в фазе 2). |
| **Tablet и desktop** (`md` и выше) | Полная сценография: колонки, видео, смена с буквами, плавные переходы. |

Точка отсечения — breakpoint **`md`** из `tailwind.config.ts` (768px), без новых произвольных пикселей в JSX.

## 3. Доступность: `prefers-reduced-motion`

- **Проверка**: `window.matchMedia('(prefers-reduced-motion: reduce)')` или CSS `@media (prefers-reduced-motion: reduce)`.
- **Поведение при reduce**: отключить **ритмичное «мигание»** и частые смены кадров; оставить **спокойный визуал**:
  - либо **одна статичная композиция**: градиент + все буквы «Вкрайности» видимы, без циклического показа видео;
  - либо **очень медленный одиночный crossfade** (длительности вынести в токены), без стробоскопического эффекта.
- **Реализация**: длительности и easing для «нормального» и «урезанного» режима — **в `tailwind.config.ts`** (как существующие `transitionDuration`, `keyframes`) и/или один хук `usePrefersReducedMotion`, без магических чисел в разметке.

## 4. Визуал: градиент и «мягкие затенения»

- **База секции**: фон из существующих сезонных токенов (`SeasonPageBackdrop`, `bg-season-page-*`, атмосфера) — не дублировать hex в компонентах.
- **Дополнительный баннерный слой** (при необходимости): один или два градиента в `theme.extend` (направление, стопы — в конфиге), чтобы связать колонки с сезоном.
- **Fade video ↔ letter**: два слоя в колонке (видео сверху, буква под ним или наоборот — решение при вёрстке); переход только **`opacity`** (и при желании лёгкий **`filter`** из токенов), **без** scale-тряски и без контрастных вспышек.
- **Затенение**: per-column `box-shadow` / внутренняя виньетка через `linear-gradient` маску — стопы и прозрачности только из темы (`colors`, `opacity-*`), чтобы соблюсти No Hardcode Policy.

## 5. Шрифт Satyr SP Basic

| Шаг | Действие |
|-----|----------|
| Файл | Исходник: [`fonts/SatyrSP Basic.otf`](fonts/SatyrSP%20Basic.otf). Для продакшена — копия в **`public/fonts/`** с **URL-safe именем** (например `satyr-sp-basic.otf`), без пробелов в пути. |
| Регистрация | `@font-face` в том же стиле, что Nord в `bootstrap-fonts.ts` (база `PUBLIC_ASSET_BASE`). |
| Константы | Новое семейство в `src/constants/fontFamilyStacks.ts` + экспорт из `fonts.ts`; комментарий, что Satyr — **только баннер «Вкрайности»**, не замена Dela Gothic One для лого в навбаре. |
| Tailwind | Новый utility, например `font-brand-banner-word` или `font-home-season-banner`, в `tailwind.config.ts` через `theme.extend.fontFamily`. |
| Вес | Уточнить по файлу OTF (обычно одно начертание); при одном весе — `font-synthesis: none` в `@layer base` для этого класса по аналогии с `.font-brand-wordmark`. |

## 6. Данные и контент (single source of truth)

- Список **букв** (строка Unicode «Вкрайности») — из **`src/constants/ui.ts`** или отдельной константы рядом с UI секций.
- Для каждого сезона — **массив из 10 записей**: путь к **WebM/MP4** (или одному файлу + таймкоды), постер при необходимости. Файлы — под `public/`, пути — в **`src/constants/images.ts`** или **`src/data/*`** в одном месте, без дублирования в компоненте.
- Качество: короткие клипы, **вручную отобранные** секунды, приглушённая насыщенность при необходимости (через оверлей из темы).

### 6.1. Формат таймкодов и комментарии к слотам

- **Канонический источник** — [`src/data/homeSeasonBannerClips.ts`](src/data/homeSeasonBannerClips.ts): ровно **10 элементов** на сезон, индекс `0…9` = буква `UI.homeSeasonBannerWordmark` с тем же индексом.
- Поля записи:
  - **`posterSrc`** — URL постера (обязателен всегда; для видео — кадр с выбранного участка).
  - **`videoSrc`** — опционально; если отсутствует, в колонке только постер (весна/лето/осень до появления mp4 в репозитории).
  - **`startSec`** — секунда начала зацикленного фрагмента в исходном файле.
  - **`durationSec`** — длина фрагмента в секундах (плеер сбрасывает `currentTime` на `startSec` по достижении конца).
- **Комментарии к слотам обязательны** при контент-проходе: перед каждым элементом массива зимы (и при добавлении видео для других сезонов) кратко указывать, **что в кадре** и **почему выбран таймкод** (читаемость в узкой вертикали, отсутствие склейки и т.д.). Так дизайн и код не расходятся с фактическими роликами.

## 7. Техническая оркестрация (план уровня архитектуры)

- **Состояние цикла**: отдельный хук, например `useSeasonBannerColumns`, или маленький state machine: фаза «показ видео» / «показ букв», тик с `requestAnimationFrame` или `setInterval` с интервалом из константы (мс в `src/constants/` + синхрон с Tailwind при анимациях).
- **Производительность**: не держать 10 тяжёлых декодеров без нужды — при реализации рассмотреть **lazy `play()`**, `preload="metadata"`, возможное **ограничение числа одновременно играющих** клипов (план Б), если профилирование покажет просадки.
- **Библиотека анимаций**: по умолчанию **CSS + React state** и токены Tailwind; при усложнении — опционально Framer Motion (оценка размера бандла). GSAP — только если понадобится единый длинный таймлайн.

## 8. Фазы внедрения (чекпоинты)

1. **Шрифт и токены** — файл в `public/fonts/`, `@font-face`, стек в Tailwind, класс для букв баннера; `tsc` чистый.
2. **Разметка колонок `md+`** — сетка 10 колонок, placeholder без видео; mobile — упрощённый блок без сетки.
3. **Градиенты и виньетки** — всё из `theme.extend`; секция визуально цельная.
4. **Видео + crossfade с буквами** — данные из `src/data` / constants; тайминги из констант.
5. **`prefers-reduced-motion`** — отдельная ветка поведения + ручная проверка.
6. **Удаление старого `backgroundImage`** из этой секции после готовности нового баннера.

**Статус реализации:** компонент [`src/components/home/HomeSeasonBanner.tsx`](src/components/home/HomeSeasonBanner.tsx) (контейнер `max-w-7xl`, фон `home-season-banner-stage`), последовательность колонок [`src/hooks/useHomeSeasonBannerSequence.ts`](src/hooks/useHomeSeasonBannerSequence.ts), клипы [`src/data/homeSeasonBannerClips.ts`](src/data/homeSeasonBannerClips.ts), тайминги [`src/constants/homeSeasonBannerAnimation.ts`](src/constants/homeSeasonBannerAnimation.ts), шрифт `public/fonts/satyr-sp-basic.otf`. Сценарий: цепочка 0→9 только медиа (первая полоска fade in с чёрного, далее удержание и **crossfade** соседних полосок в одном окне); после 9-го клипа — **все буквы** появляются и исчезают **синхронно**; на кадре слова медиа скрыты; пауза и повтор цикла.

## 9. Критерии готовности

- Нет инлайновых цветов/spacing/z-index вне темы.
- Тексты и подписи — из `src/constants/ui.ts` (или согласованного data-файла).
- Линтер и сборка без ошибок; на `md+` баннер читаемый; при включённом reduced motion — нет дискомфорта от мигания.
