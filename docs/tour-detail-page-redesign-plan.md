## 1. Нижний призыв (без изменения верхней CTA)

**Цель:** блок внизу страницы должен читаться как **заметный второй шаг** к той же заявке, а не как второстепенная «золотая кнопка как везде» (`btn-primary` сейчас совпадает с навбаром, карточками главной и т.д. — см. grep по репозиторию).

### Варианты (визуальная и структурная идея, без кода)

1. **«Завершение страницы» — карточка-полоса**  
   Нижняя зона оформляется как горизонтальный блок с фоном `surface-light` или лёгкой обводкой `border-divider`, внутренними отступами по токенам `card-p`, крупнее вертикальный ритм над/под кнопкой. Кнопка остаётся отдельным компонентом, но **контейнер** создаёт якорь внимания в конце скролла.

2. **Иерархия текста + кнопка**  
   Добавить одну строку подзаголовка из `ui.ts` (новый ключ), например пояснение «Ответим и подберём даты» — чтобы блок не был голой кнопкой. Соответствует выявленному ранее разрыву: нижний CTA визуально слабее верхнего из‑за отсутствия контекста.

3. **Визуальная связь с верхней CTA без копирования**  
   Использовать **ту же форму** (прямоугольник с `borderRadius.cta`), но **другую цветовую роль**: не оранжевый sweep (верхняя зона), а например `brand-secondary` как заливка с контрастным текстом из токенов или обводка `brand-primary` + заливка прозрачная. Так сохраняется правило «две разные эстетики CTA», но нижняя перестаёт быть клоном глобального `btn-primary`.

4. **Ширина и размер**  
   Нижняя кнопка `min-width` / padding выше, чем у стандартного `btn-primary`, выравнивание по центру или по сетке `max-w-tourDetail` с явным `max-w-*` для кнопки — чтобы на широких экранах не выглядела «потерянной точкой».

5. **Иконография**  
   Добавить иконку (FontAwesome, как в мета-бейджах) слева от подписи «Уточнить про тур» — усиление scan-линии; иконка должна дублировать смысл (вопрос / чат), не конкурировать с модалкой.

## 2. Контраст внимания: цена, сложность, время (+ синхронизация с появлением)

**Цель:** три элемента в `tour-detail-page-meta-zone` должны **первыми** удерживать взгляд после hero; сейчас цена уже на `brand-primary`, длительность — нейтральный `surface-light`, сложность — пастельные `difficulty.*`.

### Варианты

1. **Жёсткая иерархия размера и веса**  
   Цена: крупнее шаг типографики (`text-tour-detail-meta` → отдельный токен в `theme.extend.fontSize`, только для цены). Время и сложность: единый второй уровень; цена визуально доминирует — обоснование: в коммерческом лендинге цена часто главный сканируемый атрибут после заголовка.

2. **Единая «плашка-группа»**  
   Один общий контейнер с фоном `surface-light` или `color-mix` к сезону, внутри три сегмента с разделителями `divider` (не три отдельных «пилюли» на фоне страницы). Усиливает восприятие как **одного блока фактов**, а не трёх случайных чипов.

3. **Последовательное появление (staggered fade/slide)**  
   Три дочерних элемента с задержкой 50–80 ms, длительность из существующих `transitionDuration.reveal` / кастомного токена; при `prefers-reduced-motion: reduce` — мгновенное появление без анимации (как уже сделано для программы тура). Связь с п.5: **тот же easing и порядок**, что у появления заголовков секций, если вводится общая «сцена» страницы.

4. **Акцент только на цене + спокойные соседи**  
   Оставить пилюли, но усилить только цену (тень `shadow-cta` из темы уже описана для CTA — оценить перенос лёгкой тени на цену; не дублировать оранжевый верхний CTA). Время и сложность — без конкурирующей насыщенности.

5. **Иконки и подписи**  
   Добавить микро-подписи над строкой («Срок», «Уровень», «Стоимость») из `ui.ts` — улучшает сканируемость для новых пользователей; увеличивает высоту мета-зоны — проверить на мобильной высоте первого экрана после hero.

## 4. Убрать клик по hero для открытия галереи

**Суть работ:** не передавать `onOpenPhoto` (и связанные `openPhotoAriaLabel`) в `TourDetailHero` из `TourDetailPage`, пока есть отдельная галерея с плитками.  
**Побочный эффект:** строка `openHeroAria` в `ui.ts` станет неиспользуемой на странице тура — решение: удалить ключ при реализации или оставить для возможного другого использования (зафиксировать в PR).  
**Доступность:** первое фото по-прежнему доступно через галерею и `TourPhotoViewer`; при необходимости добавить в текст секции галереи явную подсказку (новый ключ в `ui.ts`), если тестирование покажет снижение discoverability.

## 5. Единая система заголовков секций (опора на паттерн «Фотогалерея»)

**Текущий разрыв (факт):** «Фотогалерея» — flex + нижняя граница + вертикальный акцент; «Об этом туре» / «Что включено» / «Программа тура» — другие правила (`tour-detail-section-heading` или `font-heading` без акцента в сайдбаре).

**Направление:** вынести **общий каркас** в один примитив (класс или компонент):

- строка: опциональный **вертикальный акцент** (как сейчас `tour-gallery-section-heading-accent`);
- заголовок `font-heading` + размер из токенов (`text-tour-detail-section` или унифицированное имя);
- нижняя линия: **один** источник — либо `border-divider`, либо `tour-detail-heading-rule`, но не смешение без причины.

**Сайдбар «Программа тура»:** либо тот же каркас в уменьшенной версии (токен высоты акцента), либо осознанное исключение с записью в `design-tokens.md` — чтобы не ломать `sticky`-карточку.

## 8. Переработка элемента-акцента (вертикальная полоска у «Фотогалерея»)

Сейчас: `w-px`, `bg-brand-secondary`, `min-h-tour-gallery-heading-accent`, скрыто на `< sm`.

### Варианты усиления / разнообразия

1. **Градиент по высоте акцента**  
   Верх `brand-secondary`, низ смесь с `brand.primary` или `transparent` — в пределах `theme.extend` (без inline-цветов). Усиливает «якорь» у заголовка без увеличения ширины.

2. **Двойная линия / скруглённый капсуль**  
   Вместо 1px — узкий `rounded-full` столбик 2–3px с тем же цветом или мягкой тенью из `boxShadow` темы; визуально ближе к современным UI-kit.

3. **Сезонный оттенок**  
   Цвет акцента из `colors.season.*` по `tour.season` (уже есть сезон на странице) — связь заголовков с контекстом тура; требует новых утилит или маппинга в константах (по правилам проекта — не хардкод в JSX).

4. **Иконка вместо полоски**  
   Минимальная иконка (компас, камера) в `text-brand-secondary` с фиксированным `aria-hidden` — риск: визуальный шум рядом с `font-heading`; подходит только если иконка едина для всех секций.

5. **Анимация появления акцента**  
   Высота от 0 до `min-h-tour-gallery-heading-accent` при входе секции в viewport — согласовать с п.2 (stagger) и с `prefers-reduced-motion`.

## Три раунда критики плана

### Раунд 1 — соответствие ограничениям и правилам репозитория

- **No hardcode:** новые цвета, отступы и радиусы только в `tailwind.config.ts` / существующих константах; подписи — только `src/constants/ui.ts`. Любой вариант с «градиентом в JSX» без токена нарушает правила — в отчёте варианты указаны как направления, реализация через тему.
- **Две CTA:** усиление нижней зоны (п.1) не должно сделать её визуально сильнее верхней оранжевой CTA без намерения: иначе нарушается иерархия, зафиксированная в продуктовом решении «главная заявка сверху».
- **Hero без клика:** потеря быстрого пути к первому фото — компенсируется галереей; для пользователей с моторными ограничениями путь длиннее (скролл). Это осознанный обмен, не «улучшение без издержек».

### Раунд 2 — доступность и восприятие

- **Анимации мета-бейджей:** при последовательном fade-in экранный диктор может получить трижды подряд обновления live region — не добавлять `aria-live` на контейнер чипов; анимация чисто визуальная.
- **Контраст:** пастельные `difficulty.*` из темы уже заданы для читаемости текста на фоне; усиление «броскости» не должно опускать контраст ниже WCAG для текста/фона — проверка в браузере после внедрения, не на уровне плана.
- **Нижний блок с подзаголовком:** дублирование смысла с модалкой — минимизировать длиной строки; иначе растёт когнитивная нагрузка.

### Раунд 3 — сопровождение и связность

- **Единая система заголовков (п.5):** затрагивает `TourDetailPage`, возможно обёртки вокруг заголовков галереи; регрессии на других страницах не ожидаются, если классы не переименованы глобально без поиска по проекту.
- **Сезонный акцент (п.8, вариант 3):** добавляет ветвление стилей по сезону — больше тестовых комбинаций (4 сезона × варианты вёрстки).
- **Специальный layout галереи `izubrinaya`:** изменение заголовка над галереей должно сохранять отступы `mb-gallery-gap` и не ломать bento-сетку — проверить на `tour.id === "winter-1"` как референс.

## Итог

План опирается на **наблюдаемые** различия в коде (`btn-primary` vs `btn-cta-tour`, два стиля заголовков, три стиля мета-чипов) и на **уже принятые** паттерны проекта (`prefers-reduced-motion`, токены в `tailwind.config.ts`). Количественных метрик (конверсия, время на странице) в репозитории нет — гипотезы об усилении внимания к цене и нижнему CTA требуют проверки после внедрения (A/B или качественный разбор).
