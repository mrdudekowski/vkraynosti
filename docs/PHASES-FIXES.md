# Фазы исправлений

Документ описывает фазы внедрения исправлений по результатам ревью (см. `REVIEW-ARCHITECTURE-UX.md`). Каждая фаза — логически связанный блок задач с чётким результатом.

---

## Сводка фаз

| Фаза | Название | Цель |
|------|----------|------|
| **1** | Обработка сбоев и 404 | Приложение не «падает» при ошибках; несуществующие URL ведут на страницу «Страница не найдена». |
| **2** | Навигация и согласованность URL | Сезон в шапке совпадает со страницей; якорные ссылки ведут на главную с учётом basePath; один источник списка сезонов в футере. |
| **3** | Контент и призывы к действию | Все строки в UI; CTA на странице тура; кнопка «Политика»; контент/пометки безопасности. |
| **4** | Доступность | Skip-link, ловушка фокуса и Escape в модалке, карусель (aria-live, клавиатура), карточка тура как ссылка. |
| **5** | Полировка | Мета для «тур не найден», приоритет загрузки героя, тач-зоны, опционально лайтбокс и иконка Max. |

## Фаза 1 — Обработка сбоев и 404

**Цель:** при любой необработанной ошибке в приложении пользователь видит запасной экран с возможностью обновить страницу; при переходе по несуществующему URL показывается страница «Страница не найдена» с переходом на главную.

### Задачи фазы 1

1. **Error Boundary (глобальный перехват ошибок)**
   - Добавить классовый компонент `ErrorBoundary`, который перехватывает ошибки в дочернем дереве и рендерит fallback UI.
   - Fallback: заголовок, краткое сообщение, кнопка «Обновить страницу» (перезагрузка).
   - Все тексты вынести в `src/constants/ui.ts` (блок `errorFallback`).
   - Обернуть дерево приложения в `ErrorBoundary` в `main.tsx` (вне или внутри провайдеров — как верхний уровень, чтобы ловить ошибки везде).

2. **Страница 404 (Not Found)**
   - Добавить страницу `NotFoundPage`: заголовок, описание, ссылка «На главную».
   - Тексты и мета (title, description) — в `ui.ts` (блок `notFoundPage`) и через `PageMeta`.
   - В роутере добавить дочерний маршрут `path: '*'` с `element: <NotFoundPage />` (последним в списке children), чтобы любой неизвестный путь показывал 404.

3. **Константы**
   - В `ui.ts`: `errorFallback: { title, message, retryButton }`, `notFoundPage: { metaTitle, heading, description, homeLink }`.

### Файлы для создания/изменения (Фаза 1)

| Действие | Файл |
|----------|------|
| Создать | `src/components/errors/ErrorBoundary.tsx` |
| Создать | `src/pages/NotFoundPage.tsx` |
| Изменить | `src/constants/ui.ts` — добавить `errorFallback`, `notFoundPage` |
| Изменить | `src/main.tsx` — обернуть приложение в `ErrorBoundary` |
| Изменить | `src/router.tsx` — добавить маршрут `*` с `NotFoundPage` |

### Критерии приёмки фазы 1

- [ ] При искусственном `throw new Error()` в любом компоненте показывается экран с сообщением и кнопкой «Обновить страницу»; по клику страница перезагружается.
- [ ] Переход по URL вида `/unknown-path` или `/tours/unknown/123` (если не совпадает с `:season/:tourId` по логике роутера) отображает страницу «Страница не найдена» с ссылкой на главную.
- [ ] В консоли нет необработанных ошибок при срабатывании Error Boundary и при отображении 404.
- [ ] Все пользовательские строки на экранах ошибки и 404 взяты из `UI`.

## Фаза 2 — Навигация и согласованность URL

**Цель:** сезон в навбаре совпадает с открытой страницей сезона; ссылки «Туры», «Команда» и т.д. с любой страницы ведут на главную с нужным якорем; футер не дублирует вручную список сезонов.

### Задачи фазы 2

1. Синхронизировать `SeasonContext` с URL: при заходе на `/tours/winter`, `/tours/spring` и т.д. вызывать `setActiveSeason(season)` (хук или effect в Layout/странице).
2. Ввести единый базовый путь (например `getBasePath()` от `import.meta.env.BASE_URL`) и формировать ссылки на главную с якорем: `getBasePath() + '/' + link.href`.
3. В Navbar и Footer для якорных ссылок использовать этот базовый путь, чтобы с внутренних страниц пользователь попадал на главную и прокручивался к секции (при переходе по Link к HOME с hash — при необходимости effect для scroll).
4. В футере рендерить ссылки на сезоны через один источник (массив сезонов + маппинг на ROUTES), без дублирования четырёх отдельных `<li>`.

### Файлы (ориентировочно)

- `src/constants/routes.ts` или новый `src/utils/pathUtils.ts` — базовый путь.
- `src/context/SeasonContext.tsx` или хук `useSyncSeasonFromPath` — синхронизация с pathname.
- `src/components/layout/Navbar.tsx`, `src/components/layout/Footer.tsx` — использование базового пути и единого списка сезонов.

## Фаза 4 — Доступность

**Цель:** есть skip-link; модалка ловит фокус, закрывается по Escape и возвращает фокус; карусель объявляет слайд и управляется с клавиатуры; карточка тура — семантическая ссылка.

### Задачи фазы 4

1. В Layout добавить skip-link (скрытый до фокуса) на `#main`; у `<main>` задать `id="main"`.
2. В `TeamMemberModal`: при открытии — фокус на первый фокусный элемент (или кнопка закрытия); focus trap; закрытие по Escape; при закрытии — возврат фокуса на вызвавший элемент.
3. В `HeroCarousel`: живой регион с номером слайда; у активной точки `aria-current="true"`; на контейнер — обработка ArrowLeft/ArrowRight и `role="region"` с `aria-label`.
4. `TourCard`: обернуть в `<Link to={...}>` (сохранив визуал и область клика), убрать `role="button"` с div.

### Файлы (ориентировочно)

- `src/components/layout/Layout.tsx`, `src/components/modals/TeamMemberModal.tsx`, `src/components/home/HeroCarousel.tsx`, `src/components/shared/TourCard.tsx`, `src/constants/ui.ts` (подписи для карусели/skip-link при необходимости).

### Задачи фазы 5

4. По возможности: лайтбокс для галереи на странице тура; замена иконки Max, если продукт не VK.
